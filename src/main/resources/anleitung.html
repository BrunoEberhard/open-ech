Mit dieser Applikation können Personen und Firmen erfasst und bearbeitet werden. Es werden dabei die Meldegründe der eCH Standards 21 und 148 berücksichtigt.

Navigation

In der linken Seite des Browser befindet sich das Navigationsmenu. Es kann über das Kreuz in ihrer Titelleiste geschlossen werden. Über das Symbol der drei Linien oben links kann es bei Bedarf wieder angezeigt werden. Dieses Menu stellt den Einstiegspunkt für eine neue Bearbeitung dar. Die aktuelle Anzeige eines Steuerauszugs wird dabei komplett ausgetauscht. Das Menu kann jederzeit benutzt werden ausser es ist gerade ein Dialog geöffnet.

Wie gewohnt kann zudem mit den Vor- und Zurück Knöpfen des Browser im Verlauf geblättert werden.

Anwendungsfälle

Die Applikation beschäftigt sich mit der Erfassung und Mutation von Personendaten. Personen können auf drei Arten erfasst werden:

  • Eine Person kann als Zuzug erfasst werden
  • Eine Geburt kann erfasst werden
  • Die Daten können von einer bestehenden Applikation übernommen werden (Basislieferung, nur möglich mit entsprechender Berechtigung)

Umgekehrt kann eine Person die Verwaltung auf folgende Art verlassen:

  • Der Wegzug der Person kann erfasst werden
  • Die Person kann gestorben sein
  • Die Person kann als verschollen gemeldet werden (das kann als einziges rückgängig gemacht werden)

Alle anderen fachlichen Anwendungsfälle beziehen sich auf einen Meldegrund. Anwendungsfälle ohne Meldegründe sind Korrekturen (also Behebungen von Fehlern).

Zuzug

Ein Zuzug kann mehrere Personen auf einmal betreffen. Wenn ein Ehepaar mit Kindern zuzieht, so sollten zuerst die Eltern und danach die Kinder erfasst werden. Damit werden die Beziehungsdaten automatisch richtig erstellt.

Bei einer Beziehung zwischen Kind/Eltern wird jeweils nur die Eltern - Beziehung erfasst. Das heisst, es exisitieren keine Eintragungen bei den Eltern.

Nach Abschluss der Eingabe der ersten Person des Zuzugs wird nachgefragt, ob eine weitere Person erfasst werden soll. Dabei muss unterschieden werden zwischen (Ehe-)Partner und Kindern:

  • Ein (Ehe-)Partner kann nur erfasst werden, wenn der Zivilstand der bereits erfassten Person den nötigen Eintrag hat. Da bei einem (Ehe-)Partner die Beziehungsdaten gegenseitig eingetragen werden, wird (im Hintergrund) mit der definitven Speichern der 1. Person zugewartet, bis auch die 2. Person abgeschlossen wird. Dieser Vorgang ist in der Regel für den Benutzer transparent, nur im Fehlerfall kann es sein, dass sich ein Fehler auf die scheinbar schon gespeicherte 1. Person bezieht. Ausserdem ist die 1. Person erst suchbar, wenn sie zusammen mit der 2. gespeichert wurde.
  • Bei Kindern wird eine Mutter- und eine Vaterbeziehung vorgeschlagen. Die Werte werden dabei von der ersten Person (links im Dialog) übernommen. Daher kann es sinnvoll sein, Mutter und Vater zu tauschen, vorgeschlagen wird aber die Mutter als Vorlage.

Geburt

Am einfachsten kann eine Geburt erfasst werden, wenn gerade ein Elternteil des Kindes angezeigt wird. In diesem Fall werden einige Angaben (z.B. Familiennamen etc.) auf die Eingabemaske übernommen. Ist eine Beziehung zu einem Ehepartner / einer Ehepartnerin vorhanden so werden auch diese Angaben übernommen.

Weitere Funktionen

Suchen

Die Suche wird jeweils als Einstieg bei jedem neuen Tab angeboten. Das Resultat der Suche wird tabellarisch angezeigt.

Versionen einer Person

Es können die verschiedenen Versionen einer Person angezeigt werden. Dies bei der Anzeige einer Person über den Menupunkt "Verlauf anzeigen". Durch Anwählen eines Eintrages in der Tabelle kann der jeweilige Stand der Daten bis und mit dem gewählten Eintrag angezeigt werden.

Hinweis: Verändert werden kann nur die aktuellste Version der Person, daher sind die Änderungsmenus im Fall von älteren Versionen deaktiviert.

Links bei Beziehungen

Wird bei einer angezeigten Person eine Beziehung angezeigt oder ein Kontakt zu einer anderen Person so kann durch ein Mausklick auf die Beziehung oder den Kontakt zur betreffenden Person navigiert werden.

Datumsfelder

Ein Datumsfeld enthält die Angaben zu Tag, Monat und Jahr. Bei Geburtstagen kann der Monat und der Tag weggelassen werden, falls diese Angaben nicht bekannt sind. Die Eingabe wird so gut wie möglich ergänzt. Die Übersetzung geschieht wie folgt:

  • 030607 wird zu 03.06.2007 also dem 3. Juni im Jahr 2010
  • 2.2.02 wird zu 02.02.2002 also dem 2. Februar 2002
  • 1.1.99 wird zu 01.01.1999 Zweistellige Jahreszahlen werden bis 20 als 20xx, ab 21 als 19xx interpretiert

Zusätzlich kann mit den Cursor/Pfeil - Tasten das Datum geändert werden. Mit Pfeil runter wird der folgende Tag gewählt, mit Pfeil rauf der vorhergehende. Mit der Umschalttaste wird das Jahr gewechselt, mit der Alt-Taste der Monat.

Login / Authorization

Mit dem Pfeil Symbol oben links kann man sich als neuen Benutzer anmelden. In der öffentlichen Installation wird auch dem anonymen Benutzer alle Möglichkeiten zur Erfassung und Bearbeitung von Personen gewährt, damit zu Beginn kein Login notwendig ist. Geschützt sind nur die administrativen Funktionen wie Import-/Export und Systemkonfiguration.

Schemaversionen

Bei der Einwohnerkontrolle (EWK) werden verschiedene Versionen der Spezifikation von Ech unterstützt. Die technische Spezifikation besteht dabei aus einem sogenannten XML - Schema. Die Version des Schema wird dabei bei jeder Anpassung von Ech hochgezählt.

Jede neue Version bietet neue Möglichkeiten. Das bedeutet meistens mehr Felder auf einer Eingabemaske. Im Normalfall kann mit dem neusten Schema gearbeitet werden. Ältere Schemas sind nur für technisch Interessierte gedacht, die damit z.B. die Kompatibilität zu früherer Software überprüfen möchten.

Jedes Fenster verwendet genau eine Schemaversion. Um ein Fenster zu öffen, das mit einer anderen Version arbeitet wählen Sie den Menupunkt "Datei/Neues Fenster mit..".
